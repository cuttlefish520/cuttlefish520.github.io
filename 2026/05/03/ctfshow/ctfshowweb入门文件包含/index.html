session漏洞利用

PHP Session是用于存储用户会话数据，但这是临时文件，请求结束就会被立即删除，默认路径/tmp/sess_xxx。

在这个过程中有一个可控的参数叫PHP_SESSION_UPLOAD_PROGRESS

例如当上传文件时，PHP会自动把上传进度写入session：

可见这样就可以在session文件中写入木马，但又因为这是临时文件，写入之后会被删除，无法访问，所以在脚本中开了多个线程不断的进行写入和读取

在脚本中write方法中**’PHP_SESSION_UPLOAD_PROGRESS’:’‘这是写入session文件的木马，在read方法中session建立会话发送post请求并保持cookies不变因为这样才能访问到session文件，cookies = {‘PHPSESSID’: sessionid}，post请求体是data = { “1”:”file_put_contents(‘/var/www/html/1.php’,’‘);”}，而url为?file=/tmp/sess_’+sessionid**这一步就是把session文件包含进来，这样第一句木马才能写入，再写入第二句木马

php://filter 流的 base64-decode 方法

php://filter/[操作类型]=[过滤器列表]/resource=[目标资源]

思路就是写入文件的shell会被加上die函数，可以用php://filter/write=convert.base64-decode对<?php die('大佬别秀了');?>在写入时会进行一次base64解码就会破坏它的结构，那就不能执行die，那我们传进去的content本身就是base64编码的那就可以写入文件
