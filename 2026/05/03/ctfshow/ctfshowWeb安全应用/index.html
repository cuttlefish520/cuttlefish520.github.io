HTTPS中间人攻击

这是一个流量题很简单，要用到Wireshark工具，这是一个专门获取流量的题目。

获取包通常有以下几个数据

列名 含义
No. 数据包的序号（从 1 开始递增）
Time 相对时间戳（从抓包开始计时，单位：秒）
Source 数据包的源 IP 地址
Destination 数据包的目的 IP 地址
Protocol 网络协议类型（TCP、TLSv1.3 等）
Lengt 数据包总长度（字节）
Info 协议交互的简要信息（TCP 标志位、TLS 握手阶段等）

TCP 协议字段详解

48908 → 443：源端口 48908（客户端临时端口） → 目的端口 443（HTTPS 服务端口）

• [SYN]：TCP 同步标志位，用于发起连接（三次握手第一步）

• Seq=0：初始序列号（ISN），客户端发送的第一个字节编号

• Win=65495：滑动窗口大小，告知对方本机可接收的字节数（流量控制）

• Len=0：TCP 载荷长度（SYN 包不携带应用数据）

• MSS=65495：最大分段大小，单次 TCP 段可传输的最大数据量（避免 IP 分片）

• SACK_PERM：允许使用选择性确认（SACK），优化丢包重传效率

• TSval=872724261：时间戳值，用于 RTT 计算和防序列号回绕

• TSecr=0：时间戳回显值（握手阶段为 0，后续 ACK 会回显对方 TSval）

• WS=128：窗口扩大因子，将 Win 字段左移 7 位（2⁷=128），实际窗口 = Win × WS

关键交互阶段解析

1. TCP 三次握手

  1. 包 1 (SYN)：客户端向服务端发起连接请求
  2. 包 2 (SYN, ACK)：服务端确认连接，并发起自己的同步请求
  3. 包 3 (ACK)：客户端确认服务端的同步，TCP 连接建立完成

2. TLS 1.3 握手

  • (Client Hello)：客户端发起 TLS 握手，携带 SNI（ctfshow.com）、支持的密码套件等
  • (Server Hello / Change Cipher Spec / Application Data)：服务端完成握手，发送证书、协商密钥，并开始加密传输
  • (Change Cipher Spec / Application Data)：客户端切换到加密状态，后续数据均为加密的 Application Data

3. 应用数据传输

  • 所有 TLSv1.3 Application Data 均为加密后的 HTTPS 流量，无法直接查看明文内容
  • 期间穿插 TCP ACK 包（如包 5、12），用于确认数据接收

4. 连接重置（包 14）

  • [RST]：重置标志位，强制关闭连接
  • 通常由服务端主动发送，原因可能是：
    • 服务端异常终止
    • 客户端请求被拒绝
    • 连接超时无响应

那TLS被加密了，无法查看，在下载的附件里还有一个sslkey.log的文件

sslkey.log

sslkey.log 通常是一个与 SSL/TLS 密钥日志相关的文件，主要用于记录 SSL/TLS 会话过程中产生的密钥信息，以便解密加密的网络流量（如 HTTPS 通信），常见于网络调试或分析场景。
￼
核心作用
SSL/TLS 协议通过加密保护网络通信（如浏览器与服务器的 HTTPS 连接），第三方通常无法直接查看加密内容。但 sslkey.log 会记录会话中的临时密钥（如 CLIENT_RANDOM 对应的主密钥），工具（如 Wireshark）可利用这些密钥解密捕获的 SSL/TLS 流量，方便开发者或分析师调试加密通信问题。

在工具左上角编辑->首选项->Protocol,在选择TLS,在(Pre)-Master-Secret log filename 配置上sslkey.log文件，然后就会多出两个http流量包，查看包内容即可

反弹shell构造

相关理论知识

什么是shell
shell是渗透中常用的名词，像getshell，webshell，反弹shell等等，都和shell相关。
• getshell：获取到目标的命令执行权限
• webshell：指网站后门，通过web服务进行命令执行
• 反弹shell：把命令行的输入输出转移到其它主机
Shell 俗称壳（用来区别于核），是指“为使用者提供操作界面”的软件（命令解析器）。它类似于DOS下的command.com和后来的cmd.exe。它接收用户命令，然后调用相应的应用程序。简单说用户通过壳（shell）访问操作系统内核的服务，也就是由壳到内核，执行系统命令。

为什么要反弹shell

通常用于被控端因防火墙受限、权限不足、端口被占用等情形。
举例：假设我们攻击了一台机器，打开了该机器的一个端口，攻击者在自己的机器去连接目标机器（目标ip：目标机器端口），这是比较常规的形式，我们叫做正向连接。远程桌面、web服务、ssh、telnet等等都是正向连接。那么什么情况下正向连接不能用了呢？
有如下情况：

  1. 某客户机中了你的网马，但是它在局域网内，你直接连接不了。
  2. 目标机器的ip动态改变，你不能持续控制。
  3. 由于防火墙等限制，对方机器只能发送请求，不能接收请求。
  4. 对于病毒，木马，受害者什么时候能中招，对方的网络环境是什么样的，什么时候开关机等情况都是未知的，
  5. webshell下执行命令不交互，为了方便提权或其它操作必须要反弹shell。
  6. 反弹shell相当于新增一个后门，当webshell被发现删除后权限不会丢失。
    所以建立一个服务端让恶意程序主动连接，才是上策。
    那么反弹就很好理解了，攻击者指定服务端，受害者主机主动连接攻击者的服务端程序，就叫反弹连接。
