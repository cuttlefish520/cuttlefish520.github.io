1.什么是java反射

官方介绍

当我们知道要实例化的类的时候，都是直接new Objectname(),这个过程是正射，那当我们不知道实例化的对象是什么的时候就要用到java反射，它可以让我在运行是动态的获得类的信息。

2.反射常用API

  • java.lang.Class：表示类的对象。提供了方法来获取类的字段、方法、构造函数等。
  • java.lang.reflect.Field：表示类的字段（属性）。提供了访问和修改字段的能力。
  • java.lang.reflect.Method：表示类的方法。提供了调用方法的能力。
  • java.lang.reflect.Constructor：表示类的构造函数。提供了创建对象的能力。

反射的基本步骤是，获得Class,在通过Class的getConstructor()和getMethod()分别获得Constructor和Method，而Constructor的newInstance()可以创建反射类的实例，Method的invoke()可以调用其方法，getFields()可以获得Field，可以修改反射类的属性变量。

2.1获取Class方式

Class类一般来说是用来获取类对象的，获取类对象的方法有五种

  • 根据类名：Class<?> clazz = 类名.class
  • 根据对象实例：Class<?> clazz = 对象.getClass()
  • 根据全限定类名：Class<?> clazz = Class.forName(全限定类名)
  • 通过类加载器：Class<?> clazz = ClassLoader.getSystemClassLoader().loadClass("java.lang.String")
  • 基本类型包装类通过静态字段 TYPE获取类对象：Class<?> clazz = Integer.TYPE

看一下getConstructor方法

  • @CallerSensitive 这是 java.lang 包的核心注解 表示这个方法的行为依赖调用者的类，JVM 会特殊处理调用栈，用来做安全校验
  • Constructor<T>：返回当前类 T 的构造器对象
  • parameterTypes：可变参数，用来指定要匹配的构造方法的参数类型列表 例如 getConstructor(String.class) 匹配参数为 String 的构造方法 getConstructor() 匹配无参构造方法
