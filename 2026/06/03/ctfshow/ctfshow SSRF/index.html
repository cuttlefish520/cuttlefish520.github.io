1.什么是SSRF

1.1 ssrf漏洞介绍

SSRF（Server-Side Request Forgery，服务端请求伪造）是一种网络安全漏洞，攻击者通过该漏洞可以利用服务器端的应用程序发起伪造的请求。

1.2 漏洞原理

当服务器在接收用户可控的URL或其他资源地址时，没有做严格的限制和过滤。服务器话可以访问这个地址获取资源，就有可能导致ssrf。

条件

  1. 用户可控的url地址
  2. 应用程序可以发起服务端请求
  3. 应用程序对请求缺乏充分的验证和限制
  4. 请求要以服务器的身份和权限去执行，可以获得内部资源

一个典型的SSRF漏洞场景：某网站提供在线图片处理功能，允许用户指定图片URL，服务器获取该图片后进行处理。如果该功能未对URL进行严格验证，攻击者可以提供指向内部服务的URL（如http://internal-service:8080/admin），服务器会尝试访问这个内部服务，从而可能泄露敏感信息或触发未授权操作。

1.3 ssrf的利用

内网访问

利用SSRF漏洞访问内网中的Web应用或其他服务。通过构造特定的URL，攻击者可以访问内网中的敏感资源。

302重定向

在网站目录下写一个php文件，url可控的情况下，传入自己的服务器，去访问php文件，触发重定向

伪协议读取文件

利用PHP伪协议读取本地文件。

1.4 一些绕过

IP转换

sudo.cc

sudo.cc是一个域名，但作用类似于127.0.0.1，即指向本地回环地址.

关于回环地址

在IPv4中，127.0.0.1~127.255.255.254都是本地回环地址的范围，有时候只针对127.0.0.1进行了过滤，还可以用其他地址.

域名解析

其实和sudo.cc是一样的，但这还有一些，大厂的域名可解析到127.0.0.1，见一些公共 http 域名，但是解析到 127.0.0.1_可以被解析为127.0.0.1的域名-CSDN博客

0在Linux系统的解析

0在linux系统中会解析成127.0.0.1，但在windows中会解析成0.0.0.0。

省略0

127.1也是可以解析成127.0.0.1

特殊符号

@：作为身份分隔符，@前面的内容都是身份认证部分比如http://acb@127.0.0.1服务器还是会访问@后面的，前面只做身份验证

# ：是片段标识符，#后面的所有内容都是客户端本地的锚点，永远不会被发送到服务器。

?： 是查询参数分隔符，**?**后面的内容是 URL 的查询参数。

解释一下出现的函数和参数

gethostbyname()

gethostbyname() 函数用于将主机名（域名）转换为对应的 IP 地址。在网络编程中，这个函数非常有用，因为它可以帮助我们通过域名获取服务器的 IP 地址，从而进行网络通信。

filter_var()

filter_var函数是PHP中用于过滤变量的功能强大的工具。它可以使用指定的过滤器来清理或验证输入数据，确保数据符合预期的格式或标准。

  • $variable必须参数，需要过滤处理的变量
  • $filter可选，过滤器id字段，默认使用FILTER_DEFAULT，相当于FILTER_UNSAFE_RAW，默认情况下不进行过滤。
  • $options可选，可做一些过滤器的配置，精细控制过滤器的行为。
