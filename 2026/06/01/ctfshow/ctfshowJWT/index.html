1.什么是JWT

JWT 是 JSON Web Token 的缩写，是一种基于 JSON 的开放标准（RFC 7519），用于在网络应用中安全地传输信息。它广泛应用于用户身份验证和授权，尤其在跨域认证场景中非常流行。

1.1 jwt的组成

jwt有三部分组成

  • Header(头部)：包含元数据，如签名算法（如 HS256）和令牌类型（JWT）。
  • Payload(载荷)：存储实际传输的数据，例如用户信息和声明（Claims）。声明分为注册声明（如 iss、exp）、公共声明和私有声明。
  • Signature（签名）：通过头部、载荷和密钥生成，用于验证数据完整性，防止篡改。用于校验

1.2 jwt的特点

  1. 无状态性：JWT 是无状态的，所有信息都包含在令牌中，服务端无需存储会话数据，减轻了服务器负担。
  2. 跨平台性：基于 JSON 格式，支持多种编程语言和平台。
  3. 安全性：通过签名保证数据完整性，可选加密保护敏感信息。
  4. 扩展性：支持自定义声明，满足不同业务需求。

1.4 JWT 缺点与注意事项

  1. Payload 可解码，不能存敏感信息（密码）
  2. 一旦签发，无法中途作废（除非用黑名单）
  3. 过期时间不能太长，否则不安全
  4. 密钥 secret 绝对不能泄露
  5. 体积比 SessionId 大，每次请求携带消耗流量

1.5JWT令牌的生成

上面说jwt的签名部分是由头部，荷载，和密钥生成，这里还分一个对称加密和非对称加密

对称加密如HS256就只需要一个密钥，生成令牌与验证都是用这一个密钥

非对称加密如RS256 / ES256用到两个密钥，一个私钥一个公钥，私钥用来生成令牌，而公钥用来验证令牌，企业多使用非对称加密，而私钥是绝对保密的

字段 全称 含义 你的示例值说明

iat Issued At 令牌签发时间 1780054750：Unix 时间戳，对应北京时间 2026-05-28 19:39:10
exp Expiration Time 令牌过期时间 1780061950：Unix 时间戳，对应北京时间 2026-05-28 21:39:10
nbf Not Before 令牌生效时间 1780054750：表示令牌从签发时刻立即生效
sub Subject 令牌主题 user：表示这个令牌是颁发给普通用户的
jti JWT ID JWT 唯一标识符 8765caf3611ea6693c525b2170244eaf：用于唯一标识这个 JWT 令牌
