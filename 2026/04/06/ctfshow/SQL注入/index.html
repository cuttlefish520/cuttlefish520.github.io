基础知识 一库三表

INFORMATION_SCHEMA: mysql默认数据库

SCHEMATA（表）：存储mysql下每一个数据库的相关信息

​ schema_name(字段)：数据库名

TABLES（表）：存储mysql下每一个表名

​ table_name（字段）：数据表名称

​ table_schema(字段)：存放表所在的数据库名

COLUMNS(表)：存放mysql下所有列的名称

​ column_name:字段名称

​ table_name:该字段所在的数据表名

​ table_schema:该字段所属的数据表所在的数据库名

这三张表在information_schema数据库下，在SQL注入过程中不能直接指定表名，要使用数据库名.数据表名，列如:information_schema.tables

基础语法

增：INSERT table_name(columns_name) VALUES(new_values)

删：DELETE table_name WHERE condition

查：SELECT columns_name FROM table_name WHERE condition

改：UPDATE table_name SET column_name=new_value WHERE condition

ORDER BY

sql注入产生原因

当Web应用向后台数据库传递SQL语句进行数据库操作的时，如果对用户输入的参数没有经过严格过虑处理，那么攻击者就可以构造特殊的sql语句，直接输入数据库引擎执行，获取或修改数据库中的数据

一些绕过

空格可用/**/,%09，%0c,%0A,%0B,%0D,()包裹标识符，--%0A代替。 注释符%23,--+,--A。 sql语法中like使用%通配符，like '匹配符%'。过滤了引号就是不能传字符，可以将字符串编码成16进制绕过。字符转换成数字开头都是0，所以可以用0弱匹配。#用0x23代替。~用0x7e。

1’ or ‘1’=1就是万能密码。

字符注入几乎一样，唯一不同的是，字符被引号包裹需要添加#或–+等符号去闭合引号，才能执行union select 语句

字符型注入

$sql = "select username,password from user where username !='flag' and id = '".$_GET['id']."' limit 1;";

通过这串代码可以看出执行的sql语句条件是username不能为flag，id通过get传参，'".$_GET['id']."'由引号包裹，判断是字符型注入，输入1' union select 1,2,3 --+，如果输入是1 union select 1,2,3就无法执行union后的语句

查库:输入 1' union select @@version_compile_os,version(),database() --+,依次返回操作系统，版本信息，当前数据库名称

报错注入

通过特殊函数的错误使用使其参数被页面输出，有点像SSTI。当然，这种注入可以成功的前提是服务器开启报错信息返回，也就是发生错误时返回报错信息，常见的利用函数有常见的利用函数有：exp()、floor()+rand()、updatexml()、extractvalue()等，

在 SQL 中，floor() 是 向下取整函数（数学功能），在 报错注入 里，它的核心作用是：配合 rand()、group by 制造 “主键冲突” 报错

group by 分组时，会先执行 floor(rand(0)*2) 生成 0/1 作为临时表主键，但 rand() 函数 每次调用都会重新生成随机数—— 导致第一次生成的主键（比如 0），第二次调用时变成了 1，临时表中出现 “重复主键”，数据库就会报错：Duplicate entry 'xxx' for key 'group_key'（键值重复）。

把 查询敏感数据的语句（比如 database()）和 0/1 拼接，让 “重复的键值” 就是敏感数据，从而在报错中泄露出来。
